1. 추상클래스 (abstract class)  // 내용없이 선언(정의)만 있음  --> 설계 모델링 하는 클래스
    1) 구현 코드 없이 메서드의 선언만 있는 추상 메서드 (abstract method)를 포함한 클래스 //객체를 생성 할 수 없음
        - 메서드 선언(declration) : 반환타입, 메서드 이름, 매개변수로 구성
        - 메서드의 정의(definition) : 메서드 구현(implementation)과 동일한 의미
                                    구현부(boby)를 가짐({})
            - int add( int x, int y);               // 선언   추상메서드
            - int add( int x, int y) {}             // 구현부가 있음. 추상 메서드가 아님     

    2) abstract 예약어를 사용
    3) 추상 클래스는 new를 할수 없음(인스턴스화 할수 없음)

2. 추상 클래스 구현하기    
    1) 메서드에 구현 코드가 없으면 abstract로 선언
    2) abstract로 선언된 메서드를 가진 클래스는 abstract로 선언
    3) 모든 메서드가 구현된 클래스라도 abstract로 선언되면 그냥 클래스가 아닌 추상 클래스로 인스턴스화(객체화) 할 수 없음
    
    4) 추상 클래스의 추상 메서드는 하위 클래스가 상속하여 구현
        -추상 클래스 내의 추상 메서드 : 하위 클래스가 구현해야 하는 메서드 
        -추상 클래스 내의 구현된 메서드 : 필요에 따라 하위 클래스에서 재정의 함
